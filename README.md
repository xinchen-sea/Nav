# 尘心 · 轻首页

页面定位是私人轻首页：搜索、常用入口、今日待办和临时记录都放在首屏内，保留轻量、直接、偏工作台的使用方式。

## 文件结构

```text
.
├── index.html      # 页面结构
├── styles.css      # scheme 视觉样式
├── app.js          # 搜索、入口筛选、待办和记录逻辑
├── assets/
│   ├── ico.png     # favicon
│   └── ico2.png    # 页面头像
└── CNAME           # GitHub Pages 自定义域名
```

## 使用

直接打开根目录的 `index.html` 即可使用。也可以部署到 GitHub Pages、Netlify、Vercel 或任意静态托管服务。

搜索框支持：

- 输入普通内容：使用当前搜索引擎搜索。
- 输入网址：直接跳转到对应网站。
- 输入常用网站名称或标记：直接打开对应入口。
- 输入 `n 想法内容`：追加到临时记录。
- 输入 `t 任务内容`：追加到今日待办。
- 按 `/`：聚焦搜索框。

当前搜索引擎包括 Bing、百度、Google、GitHub、YouTube 和 Bilibili。搜索引擎选择会保存在当前浏览器中。

## 常用入口

页面保留 37 个常用入口，并按以下分组展示：

- 校园核心
- 学习资源
- 写作文档
- 开发工具
- 休闲娱乐
- 个人相关

搜索框输入内容时，会跨全部分组实时筛选入口；清空搜索框后恢复当前分组。

## 本地保存

今日待办、临时记录和搜索引擎选择使用浏览器 `localStorage` 保存。当前版本继续使用 scheme2 的独立 key：

```text
scheme2-refined.tasks
scheme2-refined.quick-note
scheme2-refined.search-engine
```

这些数据只保存在当前浏览器中，换浏览器或清理站点数据后不会自动同步。

## 调整入口

常用网站数据位于 `app.js` 的 `LINK_GROUPS`。每个入口格式为：

```js
["标记", "名称", "链接", "关键词"]
```

修改后刷新页面即可生效。
